Navigation structure
The app has 3 main navigation areas:

Auth flow

Login
Register
Candidate app

Bottom tabs for candidate features.
Extra stack pages for offer details, AI recommendations, and chat.
Recruiter app

Bottom tabs for recruiter features.
Extra stack pages for offer creation/editing and chat.
1. Auth pages
LoginScreen
File: src/screens/auth/LoginScreen.tsx

Purpose: lets the user sign in with Firebase email/password.

Uses:

AppInput
Email input
Password input
AppButton
Sign in button
Create account button
AnimatedEntrance
Entrance animation for brand block and form
SurfaceCard
Card container for login form
UI/UX:

Dark Interlance hero background.
Interlance logo mark.
Modern welcome card.
Clear microcopy: “Where internships and freelance talent meet.”
RegisterScreen
File: src/screens/auth/RegisterScreen.tsx

Purpose: lets a user create either a candidate or recruiter account.

Uses:

AppInput
Full name
Email
Password
AppButton
Create account
Back to login
AnimatedEntrance
Animates sections into view
Local RoleCard
Candidate role selector
Recruiter role selector
UI/UX:

User chooses between Candidate and Recruiter.
Uses modern selectable role cards.
Creates Firebase account, then syncs backend user.

6. Navigation components
AppNavigator
File: src/navigation/AppNavigator.tsx

Main app router.

Logic:

If app is initializing → show LoadingView
If no user → show AuthNavigator
If user role is RECRUITER → show RecruiterNavigator
Otherwise → show CandidateNavigator
Also:

Connects/disconnects realtime client.
Registers push token.
Applies Interlance navigation theme.
